eZ component: Url, Design
~~~~~~~~~~~~~~~~~~~~~~~~~
:Author:   Alexandru Stanoi
:Revision: $Revision: $
:Date:     $Date: $

Design description
------------------

ezcUrl
   The url passed to the constructor is parsed (broken down into parts), and
   it is reconstructed with the method __toString(). A static array holds the
   path prefixes (eg. 'images', 'css'). Another static array holds the
   basedirs (along with related info like scheme, user and password). The
   basedirs can be extracted from urls.

Algorithms
----------

// register the url path '/media/pictures' as 'images'
ezcUrl::registerPath( 'images', '/media/pictures' );

// register the basedir 'www.example.com/mydir' as 'ex'
ezcUrl::registerHost( 'ex', 'http://www.example.com/mydir' );

// create a new url object
url = new ezcUrl( 'spacer.gif', 'ex' );

// append the registered path and host (in any order).
url->prefixPath( 'images' );

// or the 2 lines above can be written like:
// url = new ezcUrl( 'spacer.gif', 'ex', 'images' );

// build the url ('http://www.example.com/mydir/media/pictures/spacer.gif')
str = url->toString();
